# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제:

2인용 체스, 사용자가 직접 기물 제작 가능한

---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

내 설명: 사용자가 직접 창작한 체스 기물을 체스판 위에서 기능할 수 있게 만들고 싶다.


---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 현재 기물들의 위치.
- 사용자가 선택한 기물의 행마법과 그에 따른 상호작용.
- 새 기물의 행마법.

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 기물들의 행마법 합법 여부 검사.
- 
- 

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 각 차례 기물들의 움직임과 그에 따른 행동 결과.
- 승리조건 달성 시 게임 종료.

---

| board_now | 현재 체스판 상태 출력 | 기물의 이동 | 이동에 따른 결과 2차원 리스트 |

## 6. 사용할 제어구조

### 조건문을 사용할 부분

내 계획: 기물이 가능한 행마법과, 잡거나 잡히거나, 체크 등의 상호작용을 할 때 사용한다.


### 반복문을 사용할 부분

내 계획: 게임이 끝나기 전까지 내 차례와 상대 차례를 번갈아 반복 할 때 사용한다.
